## 一、总体设计原则

### 教学哲学（源自 teach skill）

1. **学以致用**：每节课都以"为什么学"开头，关联真实问题（estimation、状态估计、Kalman Filter）
2. **ZPD 递进**：严格按照"研究素养→线性代数基础→系统理论→估计理论→Kalman Filter→延伸"的阶梯设计
3. **知识→技能→智慧**：
   - **知识**：板书公式 + SRT 提炼 + 教材笔记
   - **技能**：课后练习（计算题 + 推导题 + 编程思考）
   - **智慧**：课堂讨论 + 实际工程案例分析 + 特邀分享
4. **拆解原则**：
   - 单节 HTML 不超过 14KB（超出则拆为双节）
   - 每节有明确的教学目标和前置依赖
   - 概念速成 → 三层公式阅读（符号→含义→直觉）→ 例子巩固 → 练习检验

### 课程体系总览

```
第一阶段（0000-0003）：研究素养与软技能（预热）
第二阶段（0004-0007）：线性代数与系统基础（KF 数学根基）
第三阶段（0008-0015）：估计理论（从贝叶斯到最小二乘）
第四阶段（0016-0018）：Kalman Filter 核心（测量更新）
第五阶段（0019-0022）：Kalman Filter 深入（时间更新、增益优化）
第六阶段（0023-0026）：特邀分享（北理工、E2Map）
第七阶段（0027-0028）：KF 延伸（动态规划与 Bellman 方程）
```

## 六、教学策略工具包（基于 teach skill 框架）

针对不同难度和类型的课程，建议使用以下教学策略：

### 1. 概念速成（用于 L1-L2 的数学基础）
- **三步法**：What（定义）→ Why（动机）→ How（计算例子）
- 每引入一个概念立即配一个数值例子

### 2. 三层公式阅读（用于 L3-L17 的公式密集型课程）
- **第一层（符号）**：读得懂每个符号代表什么
- **第二层（含义）**：知道这个公式在计算什么
- **第三层（直觉）**：能用人话解释"为什么公式长这样"

### 3. 类比法（用于抽象概念）
- 特征值 = 矩阵的"拉伸系数"
- 能控 Gramian = 输入的能量"账单"
- KF 的协方差更新 = 越看越确定
- DP 的最优性原理 = "倒着推"

### 4. 案例驱动（用于特邀分享和软技能课）
- 用一个连续的案例贯穿整节
- 练习使用案例变体（"如果⋯⋯会怎样？"）

### 5. 故事化开场（每节通用）
- 每节 Hero 标题下的 subtitle 应包含一个"一句话故事"
- 示例："从贝叶斯公式到卡尔曼滤波——一条贯穿估计理论的主线"
