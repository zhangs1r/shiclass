1. 生成式模型 × 具身智能：背景

近年来，具身智能（Embodied AI）领域涌现出一批基于大规模模仿学习的模型，以 VLA（Vision-Language-Action） 为代表。这类模型接收图像、本体状态、自然语言指令作为输入，通过训练好的编码器将观测压缩为 Token，最终输出机器人的动作。

🧠 输入（观测）

  • 图像（Image）— 视觉输入
  • 本体状态（State）— 关节角度等
  • 自然语言指令（Language）— VLA 特有

🎯 输出（动作）

  • 机器人动作序列 \$A \\in \\mathbb{R}^{D}\$
  • 例如 7 自由度机械臂 + 夹爪 → 8 维动作
  • 动作生成 = 给定条件 \$C\$，从 \$p(A|C)\$ 采样

💡 关键设定

定义条件 \$C = \\{ \\text{Image}, \\text{State}, \\text{Language} \\}\$，目标是生成动作序列 \$A = \\{a_0, a_1, \\dots, a_H\\}\$。生成式模型要建模的是条件分布 \$p(A|C)\$，而非确定性映射。

2. Behavioral Cloning 与它的致命缺陷

基础方法：Behavioral Cloning (BC)

最朴素的模仿学习方法：让网络直接监督人类示教数据中的动作。

\$$ \\mathcal{L}_{\\text{BC}} = \\mathbb{E}_{(C, A) \\sim \\mathcal{D}} \\| F(C) - A \\|^2 \$$

试图学习一个确定性的映射 \$F: C \\mapsto A\$

最优解分析

设 \$Y = F(C)\$，损失函数可以分解为：

\$$ \\mathbb{E}[\\|Y - A\\|^2] = \\|Y\\|^2 - 2 Y^T \\mathbb{E}[A|C] + \\mathbb{E}[\\|A\\|^2|C] \$$

求导令梯度为零得到：

\$$ Y^* = \\mathbb{E}[A | C] \$$

最优解是条件期望——即"平均动作"

⚠️ BC 的致命问题：多模态失败

假设人类示教数据包含两种抓杯方式：从左侧抓和从右侧抓。BC 的最优解会选择一个"中间路线"——既不像左也不像右，最终无法成功抓取。

这就是 BC 无法处理 多模态（multimodal） 动作分布的根本原因。生成式模型通过引入随机性来解决这一问题：不是在确定性的"平均"上做预测，而是从概率分布中采样。

3. DDPM（扩散模型）

Denoising Diffusion Probabilistic Model 是生成式模型在动作生成领域的开山之作之一。

核心思想

🔁 正向过程（加噪）

从干净的动作分布 \$x_0\$ 开始，逐步添加高斯噪声：

\$$ x_t = \\sqrt{\\bar\\alpha_t}\\, x_0 + \\sqrt{1 - \\bar\\alpha_t}\\,\\epsilon \$$

经过足够多步（通常 \$T=50\\sim1000\$），\$x_T\$ 接近纯高斯噪声。

🔃 反向过程（去噪）

从纯噪声 \$x_T\$ 出发，学习一个神经网络 \$\\epsilon_\\theta\$ 预测噪声，逐步去噪恢复原始动作：

\$$ x_{t-1} = \\frac{1}{\\sqrt{\\alpha_t}}\\left(x_t - \\frac{1-\\alpha_t}{\\sqrt{1-\\bar\\alpha_t}}\\epsilon_\\theta(x_t, t, C)\\right) + \\sigma_t z \$$

训练目标

\$$ \\mathcal{L}_{\\text{DDPM}} = \\mathbb{E}_{t, x_0, \\epsilon} \\left[ \\| \\epsilon - \\epsilon_\\theta(x_t, t, C) \\|^2 \\right] \$$

网络学会预测添加的噪声，而不是直接预测动作。

优势与问题

✅ 优势

  • 能够生成多模态动作分布
  • 每一步迭代都能将轨迹"拉回"合理区域
  • 理论基础扎实，变分下界可推导

❌ 瓶颈

  • 推理极慢：每步去噪都要过一遍网络
  • 典型延迟 200-300ms（50 步推理）
  • 对机器人实时控制是致命的

4. Flow Matching（流匹配）

2023 年提出的 Flow Matching 采用了一种不同的思路：不去预测噪声，而是学习一个速度场（velocity field），把噪声"流"到数据分布。

基本思想

定义一条从噪声 \$x_0 \\sim \\mathcal{N}(0, I)\$ 到数据 \$x_1 \\sim p_{\\text{data}}\$ 的概率路径：

\$$ x_t = (1 - t) x_0 + t x_1, \\quad t \\in [0, 1] \$$

线性插值路径——简单而高效

速度场就是路径对时间的导数：

\$$ \\dot{x}_t = v_t(x_t) = x_1 - x_0 \$$

目标函数

学习一个参数化的速度场 \$v_\\theta(x_t, t, C)\$ 来拟合真实速度：

\$$ \\mathcal{L}_{\\text{FM}} = \\mathbb{E}_{t, x_0, x_1} \\left[ \\| v_\\theta(x_t, t, C) - (x_1 - x_0) \\|^2 \\right] \$$

🔬 理论保证

虽然我们只能对条件概率路径（conditional path）进行监督，但论文证明：条件路径的期望梯度等于边缘路径的梯度——即学习条件路径等价于学习边缘路径。这就是 Flow Matching 的理论基础。

推理过程

通过求解常微分方程（ODE）将噪声搬运到动作分布：

\$$ x_{t+\\Delta t} = x_t + \\Delta t \\cdot v_\\theta(x_t, t, C) \$$

通常只需 10~20 步 ODE 求解 就能得到高质量结果，比 DDPM 的 50~1000 步大幅降低。

✅ 优势

  • 推理步数显著减少（10-20 步）
  • 可处理多模态分布
  • 理论优雅，实现简单

⚠️ 权衡

  • 步数越少，ODE 近似误差越大
  • 10 步 vs 100 步仍然有精度差距
  • 实时控制上仍需进一步优化

5. DDPM vs Flow Matching 对比

维度 DDPM（扩散模型） Flow Matching（流匹配）
预测目标 预测噪声 \$\\epsilon_\\theta\$ 预测速度场 \$v_\\theta\$
生成路径 随机去噪（SDE） ODE 求解
推理步数 50~1000 步 10~20 步
延迟 200~300 ms ~20 ms
多模态能力 ✅ ✅
代表应用 扩散策略（Diffusion Policy） \$\\pi_0\$、RDT 等 VLA 模型

🚀 趋势

目前 Flow Matching 已被广泛用作 VLA 模型的 Action Head，是实现高效动作生成的重要基础设施。下一讲我们将介绍如何进一步改进 Flow Matching，实现单步生成。

📝 自测题

问题 1

Behavioral Cloning 的最优解是什么？为什么这会导致多模态问题？

点击查看答案

BC 的最优解是条件期望 \$\\mathbb{E}[A|C]\$。当数据包含多个模态（如从不同方向抓杯）时，期望会输出"平均动作"，在两个模态中间摇摆不定，最终无法完成任务。

问题 2

DDPM 和 Flow Matching 的核心区别是什么？

点击查看答案

DDPM 预测噪声（\$\\epsilon_\\theta\$），通过多步去噪生成；Flow Matching 预测速度场（\$v_\\theta\$），通过求解 ODE 生成。Flow Matching 通常需要更少的推理步数（10-20 步 vs 50-1000 步）。
