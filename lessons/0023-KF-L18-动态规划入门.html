先收尾上一讲：优化视角的 Kalman gain

从抓帧可以看到，这节课开头并没有立刻写 DP，而是先把 L17 的 take-home exercise 补完：继续从二次型和完全平方角度把最优 $K_k$ 推出来。这段存在的意义，是把“估计也能看成动态优化”这个桥再搭稳一点。

$$K_k^*=P_{k|k-1}C^\top(CP_{k|k-1}C^\top+R)^{-1}.$$

随后老师才切入一个更一般的问题：如果不只是优化一次更新，而是要优化整个时域上的决策过程，该怎么办？答案就是 Dynamic Programming。

板书/抓帧

Dynamic Programming 的基本问题 P1

板书把 DP 的基本问题拆成两部分：

动态系统$$x_{k+1}=f_k(x_k,u_k,w_k)$$
可加代价$$J=\mathbb E\left[g_N(x_N)+\sum_{k=0}^{N-1}g_k(x_k,u_k,w_k)\right]$$
目标选择 $u_0,\dots,u_{N-1}$ 最小化 $J$

这就是 DP 的框架感：系统会随着时间往前走，代价也在每一步累加，所以不能只看当前一步，而要看“现在的动作会怎样影响未来”。

板书/抓帧

7-11 库存控制例子为什么经典

老师用一个单商品库存系统做例子：$x_k$ 是第 $k$ 天开始时的库存量，$u_k$ 是订货量，$w_k$ 是销售量，系统满足

$$x_{k+1}=x_k+u_k-w_k.$$

再把库存惩罚、订货成本、销售收益和终端成本合在一起，就得到完整目标函数。这个例子好在它非常生活化，但已经包含了控制问题的全部关键元素：状态、动作、噪声、滚动代价与终端代价。

板书/抓帧

从“选动作”到“选策略”

这一讲最重要的概念升级在这里。老师对比了三种情形：

  • 固定常数动作：例如每天都订 100 件。
  • 离线时间表：例如夏天订得多、冬天订得少，但与当下状态无关。
  • 闭环策略：$u_k=\mu_k(x_k)$，让动作显式依赖当前状态。
$$\pi=\{\mu_0,\mu_1,\dots,\mu_{N-1}\},\qquad \min_\pi J(x_0).$$

这一步非常关键：控制理论真正优化的通常不是某个单次动作，而是一整套策略（policy / control law）。

整理总结

为什么 KF 之后要学 DP

前面 KF 系列的重点是：在不确定观测下，怎样尽可能准确地估计状态。 现在 DP 则开始问：当状态已经在手上时，下一步该怎么决策才最好。

一句话分工：估计回答“我现在大概在哪、系统大概是什么状态”；控制回答“既然知道大概在哪，我接下来该怎么做”。

所以这节课不是另起炉灶，而是课程主线的自然延伸：先知道世界，再决定行动。
