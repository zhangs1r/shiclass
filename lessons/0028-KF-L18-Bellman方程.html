1. 回顾：动态规划的三要素

上一讲我们学习了 DP 的三个基本要素：

动态系统

$$ x_{k+1} = f_{k}(x_{k}, u_{k}, w_{k}) $$

累加代价

$$ $J = \\mathbb{E}\\{g_{N}(x_{N}) + \\sum_{k=0}^{N-1} g_{k}(x_{k}, u_{k}, w_{k})\\}$ $$

控制目标

$$ $\\min_{u_{0},\\dots,u_{N-1}} J(x_{0})$ $$

我们已经知道，闭环策略（基于当前状态决策）优于开环策略。但如何找到最好的闭环策略？这就需要引入策略（Policy）的概念和Bellman 方程。

2. 策略与控制律的形式化定义

把闭环决策规则的形式化：

$$ u_{k} = \\mu_{k}(x_{k}) $$

$\\mu_{k}$ 称为第 $k$ 步的控制律（control law）

完整的策略定义为：

$$ $\\pi = \\{\\mu_{0}, \\mu_{1}, \\dots, \\mu_{N-1}\\}$ $$

$\\pi$ 称为一个策略（policy）或控制律序列

基本问题 (P1) 最终可以表述为：

$$ \\boxed{\\min_{\\pi} J(x_{0})} $$

在所有可能的策略中，找到使总期望代价最小的那个策略

💡 理解策略

策略就是"如果看到状态 $x_{k}$，就执行动作 $\\mu_{k}(x_{k})$"的一系列规则。动态规划的目标是找到最优策略，而不是简单找到一组最优控制值。因为策略是状态到动作的映射，天然具备反馈能力。

策略空间的维度

策略空间的大小取决于状态空间和动作空间的大小。如果状态有 $|S|$ 种可能，动作有 $|A|$ 种可能，那么每个 $\\mu_{k}$ 就有 $|A|^{|S|}$ 种选择——组合数巨大！DP 的强大之处就在于不需要穷举所有策略。

3. Bellman 最优性原理

最优性原理（Principle of Optimality） 由 Richard Bellman 于 1957 年提出，是动态规划的基石：

🏆 Bellman 最优性原理

一个最优策略具有这样的性质：无论初始状态和初始决策如何，剩余的决策必须构成一个以第一个决策产生的状态为初始状态的最优策略。

直观理解

🔄 如果从 A 到 C 的最优路径经过了 B，那么从 B 到 C 的那一段也一定是最优的。

反过来说：如果从 B 到 C 不是最优的，那你就可以用从 B 到 C 的更优路径替换它，从而得到一条从 A 到 C 的更优路径——这与原来的路径是最优的矛盾。

✅ 为什么成立？

因为代价函数是可加的（分离性）。未来代价只依赖于当前状态和未来决策，不依赖于历史。这是马尔可夫性质的体现。

⚠️ 什么时候不成立？

当代价函数不可加（例如存在跨阶段的约束），或者状态不满足马尔可夫性（未来依赖于历史而非仅当前状态）时，最优性原理可能不成立。

4. Bellman 方程推导

定义最优代价函数（value function）$V_{k}(x_{k})$ 为从状态 $x_{k}$ 开始、执行最优策略到终点的最小期望代价：

$$ $V_{k}(x_{k}) = \\min_{\\pi} \\mathbb{E}\\left[ g_{N}(x_{N}) + \\sum_{i=k}^{N-1} g_{i}(x_{i}, u_{i}, w_{i}) \\Big| x_{k} \\right]$ $$

根据最优性原理，我们可以写出 $V_{k}$ 和 $V_{k+1}$ 之间的递推关系：

$$ $\\boxed{V_{k}(x_{k}) = \\min_{u_k} \\mathbb{E}_{w_k} \\big[ g_{k}(x_{k}, u_{k}, w_{k}) + V_{k+1}(f_{k}(x_{k}, u_{k}, w_{k})) \\big]}$ $$

边界条件：

$$ \\boxed{V_{N}(x_{N}) = g_{N}(x_{N})} $$

这就是 Bellman 方程——DP 的核心递推公式。

方程的含义

Bellman 方程将多步优化分解为单步优化：

  • 当前步：选择 $u_{k}$ 使 $g_{k}(x_{k}, u_{k}, w_{k})$（即时代价）+ $V_{k+1}(x_{k+1})$（未来最优代价）的期望最小
  • 未来步：$V_{k+1}$ 已经假设了从 $k+1$ 开始的所有决策都是最优的
  • 递归求解：从 $V_{N}$ 开始反向递推，直到算出 $V_{0}$

🎯 Bellman 方程的魔力

我们只需要在每个状态 $x_{k}$ 上求解一个单步优化问题，而不需要考虑未来的所有可能路径！正是最优性原理保证了这种分解是全局最优的。

5. DP 的标准求解流程

完整的 DP 求解包含四个步骤：

1️⃣ 初始化

设置终端代价函数：

$$ V_{N}(x_{N}) = g_{N}(x_{N}), \\quad \\forall x_{N} $$

为每个可能的终端状态赋值。

2️⃣ 反向递推

对 $k = N-1, N-2, \\dots, 0$：

$$ $V_{k}(x_{k}) = \\min_{u_k} \\mathbb{E}[g_{k} + V_{k+1}]$ $$

对每个可能的状态 $x_{k}$ 求解。

3️⃣ 提取策略

保存每个状态下的最优决策：

$$ $\\mu_{k}^*(x_{k}) = \\arg\\min_{u_k} \\mathbb{E}[g_{k} + V_{k+1}]$ $$

记录最优动作。

4️⃣ 正向执行

在实际运行时，根据当前状态查表执行最优动作：

$$ u_{k} = \\mu_{k}^*(x_{k}) $$

直到达到终端状态。

以库存控制为例

假设库存 $x_{k}$ 的取值范围是 0~100（离散化），订货量 $u_{k}$ 的取值范围是 0~50：

  1. 初始化：$V_{N}(x_{N}) = R(x_{N})$ 对每个 $x_{N} \\in [0,100]$
  2. 反向递推：对 $k=N-1$，对每个可能的 $x_{k} \in [0,100]$，计算 $\\min_{u_k} \\mathbb{E}[r(x_{k}) + c u_{k} - \\beta w_{k} + V_{k+1}(x_{k} + u_{k} - w_{k})]$
  3. 提取策略：对每个 $x_{k}$，记录使期望最小的 $u_{k}$
  4. 正向执行：每天根据实际库存 $x_{k}$，查表得到最优订货量

6. 维度灾难与应对思路

📌 维度灾难（Curse of Dimensionality）

DP 理论上可以解决任何离散状态和离散控制的序贯决策问题，但状态空间的维度会导致计算量指数增长。这是 DP 在实际应用中的主要限制。

假设状态是 10 维的向量，每维离散化为 10 个值，那么状态总数就是 $10^{10} = 100$ 亿——完全不可行。

常用的应对方法

📉 值函数近似

用参数化模型（如神经网络）来近似 $V_{k}(x_{k})$，而不是精确存储每个状态的值。这是深度强化学习的核心思路。

🎯 蒙特卡洛方法

通过采样而非穷举来估计期望值。只访问"高概率"的状态，忽略极不可能的状态。

🔗 问题分解

将高维问题分解为若干低维子问题。利用问题的特殊结构（如稀疏性、可分性）来降低复杂度。

⚡ LQR 特例

对于线性系统 + 二次代价 + 高斯噪声（LQG），Bellman 方程有解析解——Riccati 方程。这正是 KF 和 LQR 的联系。

7. 完整小结

本讲是 KF L18 动态规划入门系列的第二部分。总结核心要点：

概念 数学表达 核心含义
策略 $\\pi = \\{\\mu_{0}, \\dots, \\mu_{N-1}\\}$ 从状态到动作的映射序列
最优性原理 最优策略的子策略也最优 DP 可行性的理论基础
Bellman 方程 $V_{k} = \\min_{u_k} \\mathbb{E}[g_{k} + V_{k+1}]$ 分解多步为单步优化
DP 流程 初始化 → 反向递推 → 提取策略 → 正向执行 从终点倒推求解
维度灾难 状态空间指数增长 需借助近似方法

🔗 与 KF 的联系

Kalman Filter 可以看作是 DP/Bellman 方程在线性高斯系统下的一个应用。KF 的协方差更新方程（Riccati 方程）与 LQR 的 Riccati 方程互为对偶。这也是为什么我们把 DP 放在 KF 系列的最后一讲——它们是同一个理论框架的不同侧面。

📝 自测题

问题 1

Bellman 最优性原理的核心含义是什么？

点击查看答案

一个最优策略的子策略也是最优的。或者说，如果从 A 到 C 的最优路径经过了 B，那么从 B 到 C 的那一段也是最优的。这保证了多步决策问题可以分解为一系列单步优化。

问题 2

Bellman 方程 $V_{k}(x_{k}) = \\min_{u_k} \\mathbb{E}[g_{k} + V_{k+1}(f_{k})]$ 的每一项代表什么？

点击查看答案

$g_{k}(x_{k}, u_{k}, w_{k})$ 是当前步的即时代价，$V_{k+1}(f_{k}(x_{k}, u_{k}, w_{k}))$ 是从下一步状态开始的最优未来代价。期望 $\\mathbb{E}$ 对随机扰动 $w_{k}$ 取。$\\min_{u_k}$ 选择当前步的最优控制。
