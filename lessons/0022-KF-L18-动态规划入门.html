1. 什么是动态规划

动态规划（Dynamic Programming, DP） 既不是单一算法，也不是某个具体公式——它是一类解决序贯决策问题的方法论。

🧠 直觉

当你面临一系列需要先后做出的决定，而今天的决定会影响明天的局面时，动态规划就是告诉你："从最后一步往前想，每一步都选当前最好的"。

DP 的基本要素：

① 动态系统

存在一个离散时间动态系统，状态随时间演化：

\[ x_{k+1} = f_k(x_k, u_k, w_k) \]

② 代价函数

代价随时间可加（或可乘）：

\[ J = \mathbb{E}\left\{g_N(x_N) + \sum_{k=0}^{N-1} g_k(x_k, u_k, w_k)\right\} \]

③ 最优控制

目标：选择控制序列 \(u_0, \dots, u_{N-1}\) 最小化 \(J\)。

\[ \min_{u_0, \dots, u_{N-1}} J(x_0) \]

2. 基本问题 (P1) 的形式化

标准 DP 问题 (P1) 包含两个核心要素：

① 离散时间动态系统

\[ x_{k+1} = f_k(x_k, u_k, w_k), \quad k = 0, 1, \dots, N-1 \]
  • \(x_k\)：系统状态（如库存量、位置、温度等）
  • \(u_k\)：控制输入（如订货量、推力、电压等）
  • \(w_k\)：随机扰动（如需求、风速、噪声等）

② 累加代价函数

\[ J = \mathbb{E}\left\{ g_N(x_N) + \sum_{k=0}^{N-1} g_k(x_k, u_k, w_k) \right\} \]
  • \(g_N(x_N)\)：终端代价
  • \(g_k(x_k, u_k, w_k)\)：每步运行代价
  • 期望 \(\mathbb{E}\) 对随机扰动 \(w_k\) 取

求解方法：动态规划！

📌 注意

DP 的理论适用范围很广——系统可以是线性的也可以是非线性的，扰动可以是高斯的也可以是非高斯的。关键在于代价的可加性和状态的马尔可夫性（未来只依赖于当前状态和当前控制）。

3. 例子：7-11 便利店库存控制

🏪 场景设定

你是一家 7-11 便利店老板，只卖一种商品（比如可可粉）。每天你需要决定：今天订多少货？

变量定义

系统模型

\[ x_{k+1} = x_k + u_k - w_k \]
  • \(x_k\)：第 \(k\) 天开始时的库存（罐数）
  • \(u_k\)：当天订货量（罐数）
  • \(w_k\)：当天销售量（随机需求）

代价函数

\[ J = \mathbb{E}\left\{ R(x_N) + \sum_{k=0}^{N-1} \big[ r(x_k) + c u_k - \beta w_k \big] \right\} \]
  • \(r(x_k)\)：库存惩罚（如 \(r(x_k) = |x_k|^2\)）
  • \(c u_k\)：采购成本
  • \(\beta w_k\)：销售收入
  • \(R(x_N)\)：终端成本

问题解读

  • 库存过多：资金占用、仓储成本（\(r(x_k)\) 惩罚大）
  • 库存过少：缺货损失（销售机会流失）
  • 订货量 \(u_k\)：需要平衡库存惩罚和采购成本
  • 随机需求 \(w_k\)：每天卖多少不确定，需要做概率性的决策
目标：选择订货策略 \(u_0, u_1, \dots, u_{N-1}\) 最小化总期望成本 \(J\)

4. 开环控制 vs 闭环控制

同样是"决定每天订多少货"，可以有不同的决策方式：

🔓 Off-line 开环策略

提前预定好每天的订货量，不考虑实际销售情况。

例 1：\(u_k = 100, \forall k\)（每天固定订 100 罐）

例 2：根据季节预设

\[ u_k = \begin{cases} 200, & \text{春秋} \\ 500, & \text{夏天} \\ 50, & \text{冬天} \end{cases} \]

⚠️ 如果某天卖得多导致库存为负，也无能为力

🔒 On-line 闭环策略

根据当前实际库存 \(x_k\) 来决定今天订多少。

\[ u_k = \mu_k(x_k) \]

例如："如果库存低于 50 就订 200，否则订 50"

✅ 可以根据实际情况动态调整，更鲁棒

为什么闭环更好？

开环：\(u_k\) 只依赖于时间 \(k\)（\(\{u_0, u_1, \dots, u_{N-1}\}\) 提前固定）

闭环：\(u_k\) 依赖于当前状态 \(x_k\)（\(u_k = \mu_k(x_k)\)）

在存在随机扰动的情况下，闭环策略可以利用反馈信息来纠正偏离，因此通常优于开环策略。这正是控制理论的核心思想之一。

5. 策略与控制律

把闭环决策规则的形式化：

\[ u_k = \mu_k(x_k) \]

\(\mu_k\) 称为第 \(k\) 步的控制律（control law）

完整的策略定义为：

\[ \pi = \{\mu_0, \mu_1, \dots, \mu_{N-1}\} \]

\(\pi\) 称为一个策略（policy）或控制律序列

基本问题 (P1) 最终可以表述为：

\[ \min_{\pi} J(x_0) \]

在所有可能的策略中，找到使总期望代价最小的那个策略

💡 理解策略

策略就是"如果看到状态 \(x_k\)，就执行动作 \(\mu_k(x_k)\)"的一系列规则。动态规划的目标就是找到最优策略，而不是简单找到一组最优控制值。因为策略是状态到动作的映射，天然具备反馈能力。

6. DP 的核心：最优性原理

最优性原理（Principle of Optimality） 由 Richard Bellman 提出，是动态规划的基石：

🏆 Bellman 最优性原理

一个最优策略具有这样的性质：无论初始状态和初始决策如何，剩余的决策必须构成一个以第一个决策产生的状态为初始状态的最优策略。

翻译成更直观的说法：

🔄 如果从 A 到 C 的最优路径经过了 B，那么从 B 到 C 的那一段也一定是最优的。

Bellman 方程

定义最优代价函数（value function）\(V_k(x_k)\) 为从状态 \(x_k\) 开始、执行最优策略到终点的最小期望代价：

\[ V_k(x_k) = \min_{u_k} \mathbb{E}_{w_k} \big[ g_k(x_k, u_k, w_k) + V_{k+1}(f_k(x_k, u_k, w_k)) \big] \]

边界条件：

\[ V_N(x_N) = g_N(x_N) \]

DP 的求解流程

  1. 从后往前：从终端时刻 \(N\) 开始，反向递推
  2. 逐阶段求解：对每个阶段 \(k\)，对每个可能的状态 \(x_k\)，计算 Bellman 方程中的最小值
  3. 记录策略：保存每个状态下的最优决策 \(\mu_k^*(x_k)\)
  4. 正向执行：在实际运行时，根据当前状态查表执行最优动作

📌 维度灾难（Curse of Dimensionality）

DP 理论上可以解决任何离散状态和离散控制的序贯决策问题，但状态空间的维度会导致计算量指数增长。这是 DP 在实际应用中的主要限制。

📝 自测题

问题 1

动态规划解决的基本问题 (P1) 的两个核心要素是什么？

点击查看答案

① 一个离散时间动态系统 \(x_{k+1} = f_k(x_k, u_k, w_k)\)；② 一个累加的代价函数 \(J = \mathbb{E}\{g_N(x_N) + \sum g_k(x_k, u_k, w_k)\}\)。目标：选择控制序列最小化期望代价。

问题 2

在 7-11 库存控制例子中，\(x_{k+1} = x_k + u_k - w_k\) 的含义是什么？

点击查看答案

\(x_k\) 是第 \(k\) 天开始时的库存量，\(u_k\) 是当天订货量，\(w_k\) 是当天销售量。今天的库存 + 订货 - 销售 = 明天的库存。这是一个经典的状态演化方程。
